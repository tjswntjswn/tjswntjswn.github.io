개발 공부를 시작하며 가장 먼저 마주하는 **Git**과 **GitHub**.
오늘은 GitHub 공식 문서를 기반으로 두 가지의 개념과 차이점, 그리고 협업 방식에 대해 정리해보았습니다.

> **참고 문서**: [GitHub 공식 가이드 - GitHub 및 Git 정보](https://docs.github.com/ko/get-started/start-your-journey/about-github-and-git)

## 1. GitHub란 무엇인가?
**GitHub**는 코드를 저장하고, 공유하며, 타인과 함께 협업할 수 있는 **클라우드 기반 플랫폼**입니다. 쉽게 말해, 개발자들을 위한 SNS이자 클라우드 저장소라고 볼 수 있습니다.

### 주요 기능
* **리포지토리(Repository):** 코드를 저장하는 공간입니다. 여기에 코드를 올리면 작업을 쇼케이스하거나 공유할 수 있습니다.
* **이력 관리:** 시간이 지남에 따라 코드의 변경 내용을 추적하고 관리합니다.
* **코드 리뷰:** 다른 사용자가 내 코드를 검토하고 개선 제안을 할 수 있습니다.
* **협업:** '병합(Merge)' 전까지는 원본 코드에 영향을 주지 않으므로, 안심하고 여러 사람이 동시에 프로젝트를 진행할 수 있습니다.

이 모든 기능은 **Git**이라는 오픈 소스 소프트웨어를 기반으로 작동합니다.

<br>

## 2. Git이란 무엇인가?
**Git**은 파일의 변경 내용을 지능적으로 추적하는 **버전 관리 시스템(VCS)**입니다. 혼자 작업할 때도 유용하지만, 여러 사람이 하나의 파일을 동시에 수정해야 할 때 진가를 발휘합니다.

### Git의 핵심 워크플로
1. **브랜치(Branch/분기) 생성:** 원본 파일(기본 사본)에서 '가지'를 쳐서 나만의 작업 공간을 만듭니다.
2. **독립적 편집:** 내 브랜치에서 안전하게 코드를 수정합니다. 원본에는 영향을 주지 않습니다.
3. **병합(Merge):** 작업이 끝나면 Git이 변경 내용을 원본에 지능적으로 합칩니다.
4. **추적(Tracking):** 프로젝트의 최신 상태와 변경 이력을 계속해서 기록합니다.

<br>



## 3. Git과 GitHub는 어떻게 함께 작동하나요?
내 컴퓨터(로컬)와 GitHub(원격)는 **동기화**를 통해 연결됩니다.

* **로컬 작업:** 내 컴퓨터에서 파일을 수정하고 commit하면 Git이 변경 내용을 추적합니다.
* **원격 저장:** 이를 GitHub의 원격 리포지토리(Remote Repository)에 push하여 저장합니다.

### 협업 시나리오
팀 프로젝트를 진행한다면 아래와 같은 흐름으로 작업하게 됩니다.
1. **Pull:** 동료가 수정한 최신 내용을 GitHub에서 내 컴퓨터로 당겨옵니다.
2. **Work:** 작업을 진행합니다.
3. **Push:** 내 작업물을 GitHub로 보냅니다.
4. **Pull Request (PR):** 변경 내용을 합쳐달라고 요청하면, Git이 충돌을 방지하며 지능적으로 병합을 돕습니다.

<br>

## 4. 시작하기 (Getting Started)
GitHub와 Git이 처음이라면 아래 단계부터 시작하는 것이 좋습니다.

1. **계정 생성:** GitHub에 가입합니다.
2. **GitHub Flow 학습:** 브랜치, 커밋, PR, 병합 등 협업의 핵심 원칙을 익힙니다.
3. **프로필 설정:** 관심사와 기술 스택으로 프로필을 꾸밉니다.
4. **탐색 및 업로드:** 다른 사람의 오픈 소스 프로젝트를 구경하거나, 내 프로젝트를 업로드해 봅니다.
