지난 포스팅에서는 GitHub의 개념과 협업 흐름(Flow)을 배웠습니다.
그렇다면 실제로 작업을 할 때 어떤 프로그램을 써야 할까요?

GitHub는 초보자부터 전문가까지, 그리고 간단한 수정부터 복잡한 프로젝트까지 모두 아우를 수 있는 다양한 **'연결 도구'**를 제공합니다.
나의 상황과 레벨에 딱 맞는 도구를 찾아보세요.

<br>

## 1. "설치 없이 가볍게" - 브라우저 기반 도구
별도의 프로그램 설치 없이 웹 브라우저(Chrome, Safari 등)에서 바로 작업하고 싶을 때 유용합니다.

### ① GitHub.com (웹사이트 직접 수정)
* **특징:** 가장 빠르고 직관적입니다. 로그인만 되어 있다면 바로 수정 가능합니다.
* **추천 상황:** 오타 수정, 문서(Readme) 업데이트 등 단일 파일을 간단히 수정할 때.
* **장점:** `git add`, `commit` 명령어를 몰라도 UI 버튼으로 해결됩니다.

### ② github.dev (웹 기반 편집기)
* **마법의 단축키:** 리포지토리 화면에서 **마침표 키(` . `)**를 눌러보세요.
* **특징:** VS Code와 거의 동일한 화면이 브라우저에 열립니다. 파일을 이동하거나 여러 파일을 동시에 수정할 수 있습니다.
* **추천 상황:** 설치는 귀찮지만, 코드 에디터의 편리한 기능(검색, 구문 강조)이 필요할 때.

### ③ GitHub Codespaces
* **특징:** 클라우드에 있는 '고성능 컴퓨터'를 빌려 쓰는 개념입니다. 터미널 사용, 코드 빌드, 테스트까지 가능합니다.
* **추천 상황:** 내 컴퓨터 사양이 낮거나, 아이패드 같은 태블릿으로 본격적인 코딩을 해야 할 때.

<br>

## 2. "강력하고 자유롭게" - 데스크톱 기반 도구
본격적인 프로젝트를 진행한다면 내 컴퓨터(로컬)에 환경을 세팅하는 것이 좋습니다. 인터넷 없이도 작업할 수 있고, 컴퓨터의 성능을 온전히 사용할 수 있기 때문입니다.

### ④ GitHub Desktop (GUI 클라이언트)
* **특징:** 복잡한 명령어를 몰라도 마우스 클릭으로 Git을 다룰 수 있는 시각적 도구입니다.
* **추천 상황:** 검은 화면(터미널)이 무서운 입문자, 변경된 코드를 눈으로 확인하며 관리하고 싶은 분.

### ⑤ Command Line (Git & GitHub CLI)
* **특징:** 터미널에 직접 명령어를 입력하여 제어합니다. 가장 강력하고 세밀한 조작이 가능합니다.
* **추천 상황:** Git의 모든 기능을 자유자재로 쓰고 싶은 개발자, 자동화 스크립트를 짜고 싶은 분.

### ⑥ VS Code (통합 편집기)
* **특징:** 코드를 짜는 '에디터' 안에서 Git 명령어까지 한 번에 실행합니다.
* **추천 상황:** 코딩과 버전 관리를 창 이동 없이 한 곳에서 끝내고 싶은 효율성 중시형 개발자.

<br>

## 3. 한눈에 보는 도구 비교

| 도구 | 환경 | 추천 상황 |
| :--- | :---: | :--- |
| **GitHub.com** | 웹 | 아주 간단한 오타 수정, 문서 편집 |
| **github.dev** | 웹 | 설치 없이 VS Code 환경에서 코드 수정 |
| **Codespaces** | 웹 | 클라우드 컴퓨터로 빌드/테스트까지 필요할 때 |
| **GitHub Desktop** | 데스크톱 | Git 명령어가 어려운 입문자 (시각적 UI) |
| **VS Code** | 데스크톱 | 코딩과 Git 관리를 한 프로그램에서 해결 |
| **CLI** | 데스크톱 | 키보드로 모든 것을 제어하는 숙련자 |
